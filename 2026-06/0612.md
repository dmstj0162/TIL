## JPQL(Java Persistence Query Language)

- 엔티티 객체를 중심으로 개발할 수 있는 객체 지향 쿼리이다. SQL보다 간결하며 특정 DBMS에 의존하지 않는다. 방언을 통해 해당 DBMS에 맞는 SQL을 실행하게 된다. JPQL은 find() 메소드를 통한 조회와 다르게 항상 데이터베이스에 SQL을 실행해서 결과를 조회한다.

## @ManyToOne

- 여러 메뉴가 하나의 카테고리를 바라봄
- Menu 안에 Category 있음

## @OneToMany

- 하나의 카테고리가 여러 메뉴를 가짐
- Category 안에 List<Menu> 있음

## 양방향

- Menu도 Category를 알고
- Category도 Menu 목록을 앎

## 연관관계 주인

- FK를 가진 쪽
- 보통 Many 쪽
- 여기서는 Menu

@Repository
public class ParameterBindingRepository {

    @PersistenceContext
    private EntityManager entityManager;
    public List<Menu> selectMenuByBindingName;

    // 이름 바인딩 : 가독성 좋음, 실수 적음
    public List<Menu> selectMenuByBindingName(String menuName) {
        String jpql = "SELECT m FROM Section02Menu m WHERE m.menuName = :menuName";
        List<Menu> resultMenuList = entityManager.createQuery(jpql, Menu.class)
                .setParameter("menuName", menuName)
                .getResultList();

        return resultMenuList;
    }

    // 위치 바인딩 : 짧지만, 번호가 만하지면 헷갈림
    public List<Menu> selectMenuByBindingPosition(String menuName) {
        String jpql = "SELECT m FROM Section02Menu m WHERE m.menuName = ?1";
        List<Menu> resultMenuList = entityManager.createQuery(jpql, Menu.class)
                .setParameter(1, menuName)
                .getResultList();

        return resultMenuList;
    }
}
