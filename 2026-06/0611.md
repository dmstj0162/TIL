## Entity 상태 4개
1. 비영속(Transient)
- 그냥 객체, DB 모름
2. 영속(Managed)
- JPA가 관리 시작
3. 준영속(Detached)
- JPA가 관리 포기
4.삭제(Removed)
- 삭제 예정 상태 commit 시 Delete 실행

## JPQL
- SQL은 테이블 이름 사용, 컬럼 이름 사용하지만
- JPQL은 Entity 이름 사용, 필드 이름 사용

## 의미
1. Entity = DB 테이블과 연결된 클래스
2. EntityManager = JPA 관리자, JDBC Connection 같은 관리자
3. Persistence Context = JPA가 객체 보관하는 공간
4. Persist = 영속 상태로 등록, INSERT
5. Dirty Checking = 수정 감지
6. JPQL = 객체 대상으로 조회하는 SQL 비슷한 언어
7. Repository = JPA용 DAO 느낌
8. @Id = 이 컬럼이 PK이다 -> JPA는 PK가 없으면 못 움직임
9. find() = SELECT
