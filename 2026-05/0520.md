## Network 통신

- 서버는 특정 서비스를 제공하는 컴퓨터를 말하며, 클라이언트는 해당 서비스를 이용하는 사용자를 의미한다.

## WAS(Web Application Server)

- 사용자가 요청한 서비스의 결과를 스크립트 언어 등으로 가공하여 생성한 동적인 페이지를 사용자에게 보여주는 역할을 한다.

## CORS

- 브라우저가 다른 주소의 서버에 요청할 때, 서버가 허락했는지 검사하는 보안 규칙

## Servlet

- 웹서버/Tomcat 안에서 `브라우저 요청을 받아서 Java 코드로 처리하고 응답을 돌려주는 클래스`

<pre>

    브라우저 / fetch / form
            ↓ 요청
    Tomcat 서버
            ↓ 어떤 URL인지 확인
    Servlet 실행
            ↓ Java 코드 처리
    응답 HTML / text / JSON
            ↓
    브라우저가 결과 받음

</pre>

1. main()이 없는데 실행됨

- Servlet은 직접 실행하는게 아니라 Tomcat이 대신 실행해줌

<code>

    doGet(), doPost() → Tomcat이 요청 들어오면 대신 호출해주는 메소드

</code>

2. @WebServlet("/json")

- `/json` 주소로 요청이 오면 이 Servlet 클래스를 실행해라 는 의미

3. HttpServletRequest req

- 여기서 req는 요청 정보가 들어있는 객체
- 예) 사용자가 입력한 값, 요청 주소, 요청방식 GET/POST, JSON body, 파라미터, 헤더 ...

<code>

    BufferedReader reader = req.getReader();
    // 요청 body에 담겨서 온 JSON 문자열을 읽겠다

</code>

- doGet, doPost 같은 메소드에 전달되는 요청 객체

4. HttpServletResponse resp

- resp는 반대로 응답을 만들어서 보내는 객체

<code>

    resp.setContentType("application/json");
    resp.setCharacterEncoding("UTF-8");
    resp.getWriter().print(result);
    // 브라우저야, 내가 JSON 형태로 UTF-8 인코딩해서 결과 보내줄게

</code>

Servlet은 브라우저가 보낸 요청을 Tomcat이 받아서, Java 코드로 처리하게 해주는 웹 전용 Java 클래스

`웹 요청 <-> Tomcat <-> Java 코드`

- 이 사이에서 Java코드로 요청/응답을 처리하는 역할
