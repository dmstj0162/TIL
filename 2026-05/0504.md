## 추상화(Abstraction)

- 복잡한 내부 구현은 숨기고, 필요한 기능만 보여주는 것
- 왜 추상화를 쓰냐
  - 자동차 운전할 때, 엔진 구조 몰라도 "핸들, 브레이크, 엑셀"만 알면 됨
    => 이게 추상화
    => 사용자는 '어떻게 동작하는지' 몰라도 '무엇을 하는지'만 알면 됨

## DTO(Data Transfer Object)

- 데이터를 옮기기 위한 객체
- 즉, 값만 담아서 전달하는 용도로 쓰는 클래스
- 왜 DTO를 쓰냐?
- DB에서 가져온 회원 정보가 있음
- 근데 프론트(화면)에는 비밀번호는 안 보여줘야 함
  - 이 때 DTO를 쓰면, 필요한 데이터만 골라서 전달 가능

## 생성자(Constructor)

- new 연산자를 통해 객체가 메모리(heap)에 생성될 때, 가장 먼저 단 한 번 호출되는 초기화 메소드
- 객체의 필드를 원하는 값으로 '초기화'

* 생성자 작성 규칙

1. 이름이 클래스명과 반드시 동일해야 한다.
2. 반환타입(void, int 등)을 쓰지 않는다.

## 메서드 시그니처

- 메서드를 식별하는 고유한 바코드 = '메서드 이름' + '매개변수 타입, 개수, 순서'

## 오버로딩

- 이름은 같지만 시그니처가 다른 메서드를 여러개 정의하는 기술
- 불성립

1. 반환 타입만 다른 경우(시그니처 동일 -> 컴파일 에러)
2. 접근 제한자만 다른 경우(시그니처 동일)
3. 매개변수 '이름'만 다른 경우(시그니처 동일)

## 파라미터

1. 기본 자료형: '값'이 복사되어 넘어간다 --> 원본 불변
2. 배열(참조 자료형): '주소값'이 복사 되어 넘어간다 -> 원본 변경됨
3. 객체(참조 자료형): '주소값'이 복사되어 넘어간다 -> 원본 변경됨
4. 가변인자: 파라미터 개수가 유동적일 때 사용한다.

- toppings: 0개 이상의 선택 파라미터, 메소드 내부에서 배열로 취급된다.
