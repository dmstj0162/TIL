## 2차원 배열

- 디자일 배열
- 2차원 이상의 배열을 의미하며, 배열의 인덱스마다 또 다른 배열 주소를 보관하는 배열이다.
- 즉, 2차원 배열은 1차원 배열 여러개를 하나로 묶어 관리하는 배열이다.

<pre><code>
// 2차원 배열 선언 및 할당

## 얕은 복사

- stack의 주소값만 복사하므로, 두 변수가 동일한 배열(heap)을 참조한다.
- 따라서 한쪽에서 배열을 수정하면 다른 쪽에도 영향이 간다.

<code>

        // 원본 배열
        int[] originArr = {1, 2, 3, 4, 5};

        int[] copyArr = originArr;

        // hashCode()를 통해 두 변수가 같은 주소를 가리키는지 확인
        System.out.println(originArr.hashCode());
        System.out.println(copyArr.hashCode());

        System.out.println(originArr[4]);
        copyArr[4] =90;
        System.out.println(originArr[4]);

</code>

## 깊은 복사

- heap에 새로운 배열을 생성하고, 기존 배열의 값을 그대로 복사한다.
- 따라서 두 배열은 서로 독립적이므로, 한 쪽을 수정해도 다른 쪽에 영향이 없다.

<code>

int[] originArr = {1,2,3,4,5};
System.out.println("originArr : " + originArr.hashCode());
System.out.println("------------------");

        // 1. for문을 이용한 수동 복사
        int[] copyFor = new int[originArr.length];
        for(int i = 0; i < originArr.length; i++) {
            copyFor[i] = originArr[i];
        }
        print("copyFor", copyFor);

        // 2. Arrays.copyOf() 이용
        // Arrays.copyOf(원본 배열, 복사할 길이);
        int[] copyOf = Arrays.copyOf(originArr, originArr.length);
        print("copyOf", copyOf);

        // 3. System.arraycopy()
        int[] arrayCopy = new int[originArr.length];
        //System.arrayCopy(원본, 원본시작위치, 사본, 사본시작위치, 복사할 길이);
        System.arraycopy(originArr, 0, arrayCopy, 0, originArr.length);
        print("arrayCopy", arrayCopy);

        // 4. clone()

        * 참조형(객체, 2차원배열 등)에서는 얕은 복사이다.
        * */
    }

## 선택 정렬

- 가장 작은 값을 찾아 맨 앞으로 보내는 과정을 반복하는 것

<code>

        int[] arr = {2,5,4,6,1,3};
             System.out.println("정렬 전: " + Arrays.toString(arr));

        for(int i = 0; i < arr.length -1; i++){
            //반복 횟수를 -1로 설정하는 이유는 마지막 자리는 자동으로 정렬되기 때문임!!
            int minIndex = i; // 일단 현재 자리(i)에 있는 값이 가장 작다고 가정
            for(int j = i+1; j < arr.length; j++){
                if(arr[minIndex] > arr[j]){
                    minIndex = j;
                }
            }
            int temp = arr[minIndex];
            arr[minIndex] = arr[i];
            arr[i] = temp;

            System.out.println((i + 1) + "회차 정렬 후 : " + Arrays.toString(arr));
        }
        System.out.println("정렬 후 : " + Arrays.toString(arr));

</code>

## 버블 정렬

- 인접한 두 원소를 비교하여, 큰 값을 오른쪽으로 계속해서 이동시킨다.
- 한 회차가 끝나면 가장 큰 값이 배열의 맨 뒤에 위치하게 된다.

<code>

    int[] arr = {2,7,3,6,1,4};

        // 1. 정렬이 확정된 영역의 시작 인덱스(뒤에서부터 시작)
        for(int i = arr.length -1; i>= 0; i-- ){
            // j : 비교할 대상의 인덱스(앞에서부터)
            for(int j = 0; j <i; j++) {
                if(arr[j] > arr[j + 1]) {
                    int temp = arr[j];
                    arr[j] = arr[j + 1];
                    arr[j + 1] = temp;
                }
            }
            System.out.println((arr.length - i) + "회차 정렬 후 : " + Arrays.toString(arr));
        }
        System.out.println("정렬 후 : " + Arrays.toString(arr));

</code>

## 접근제한자

- 클래스 혹은 클래스의 멤버(필드)에 참조 연산자로 접근할 수 있는 범위를 제한하기 위한 키워드

- (+) public : 모든 패키지에서 접근 허용
- (#) protected : 동일 패키지에서 접근 허용(단, 상속 관계에 있는 경우 다른 패키지에서도 접근 허용)
- (~) default : 동일 패키지 내에서만 접근 허용(작성하지 않는 것이 default)
- (-) private : 해당 클래스 내부에서만 접근 허용

- 클래스, 생성자, 클래스의 멤버(필드, 메소드)에 모두 사용 가능하다.
- 단, 클래스 선언 시 사용하는 접근제한자는 public, default만 사용 가능하다.

## 캡슐화

- public 메소드를 이용해 접근하여 사용할 수 있도록 만든 기술이다.
- 클래스 작성 시 특별한 목적이 아닌 이상 캡슐화가 기본 원칙으로 사용되고 있다.

## Getter, Setter

- Getter(접근자) : 내부 필드의 값을 읽어서 전달하는 역할을 하는 메소드(읽기 전용)
- Setter(변경자) : 외부에서 전달된 값을 받아, 내부 필드의 값을 설정 또는 변경하는 역할을 하는 메소드
