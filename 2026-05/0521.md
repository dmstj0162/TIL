## servlet + JDBC

- 브라우저/클라이언트가 JSON으로 요청을 보내면 -> Servlet이 요청을 받고 -> Java 코드에서 JDBC로 DB에 접근해서 -> 조회/등록한 결과를 다시 JSON으로 응답하는 흐름

<pre><code>
                클라이언트
                ↓ JSON 요청
                Servlet
                ↓ 요청 분석
                Service
                ↓ 실제 기능 처리
                DAO
                ↓ SQL 실행
                DB
                ↓ 결과 반환
                DAO
                ↓ DTO로 변환
                Service
                ↓ Servlet으로 반환
                Servlet
                ↓ JSON 응답
                클라이언트
</code></pre>

1.  Servlet : 웹 요청을 받는 Java 클래스
    - Get 요청이면 목록 조회
    - Post 요청이면 메모 등록
    - 요청 데이터가 JSON이면 JSON으로 읽기
    - 응답도 JSON으로 보내기

2.  JSON은 '데이터 주고 받는 형식'
    - 예를 들어 메모 등록할 때 클라이언트가 이런 데이터를 보냄
      <code>
            {
                "content": "오늘 배운 내용 복습하기"
            }
        </code>
    - Servlet은 이 JSON을 읽어서 Java 객체나 문자열로 바꿔
    - 그리고 DB에 저장한 뒤 다시 이런식으로 응답
      <code>

            {
                "memoId": 1,
                "content": "오늘 배운 내용 복습하기",
                "createdAt": "2026-05-21 15:30:00"
            }

        </code>

    - 즉, JSON은 프론트와 백엔드가 대화할 때 쓰는 데이터 형식

3.  JDBC는 'Java랑 DB 연결하는 도구'

- Servlet만 있으면 요청은 받을 수 있는데, DB에 저장은 못함
- 그래서 JDBC를 사용해서 Java 코드에서 DB 연결함
  <code>

        Connection con = getConnection();
        // DB랑 연결하는 코드

        PreparedStatement pstmt = con.prepareStatement(query);
        // 이건 SQL문을 실행할 준비를 하는 코드

        pstmt.executeUpdate();
        // INSERT, UPDATE, DELETE 같은 DB 변경 작업 실행

        pstmt.executeQuery();
        // SELECT 조회 실행

    </code>

4. DAO는 'SQL 담당 클래스'

- Java 코드에서 SQL을 실행하는 곳

5. DTO는 '데이터 담는 상자'

- DB 결과나 요청/응답 데이터를 담는 객체

## 정리

1. 클라이언트가 JSON으로 요청을 보냄
2. Servlet이 요청을 받음
3. Servlet이 Service 또는 DAO를 호출함
4. DAO가 JDBC로 DB에 연결함
5. SQL 쿼리를 실행
6. DB 결과를 DTO에 담음
7. DTO를 JSON으로 바꿔서 응답함
