## enum을 사용하는 이유

- 문제상황

1. 값이 같아서 다른 과목인데 "같다"라고 판단
2. 타입 안정성 없음
3. 이름이 겹칠 경우 접두어를 붙여야해서 코드가 지저분해짐

=> 해당 문제들을 해결하기 위해 `enum` 사용

1. 타입이 달라서 비교 자체가 의미있음
2. 타입 안정성 - Subjects 외의 값은 컴파일 에러
3. values()로 전체 순회 가능

### 예시

<code>
    GUEST("게스트"),
    CONSUMER("구매자"),
    PRODUCER("판매자"),
    ADMIN("관리자");

    // 데이터를 저장할 필드 선언

    private final String description;

    // enum 생성자는 외부에서 new로 호출할 수 없기 때문에, 접근제한자는 자동으로 private이 된다.

    private UserRole(String description){
        System.out.println("UserRole 생성자 호출 " + description);
        this.description = description;
    }

    public String getDescription() {
        return this.description;
    }

    public String getNameToLowerCase(){
        return this.name().toLowerCase();
    }

</code>

## @FunctionalInterface

- 함수형 인터페이스: 메소드가 단 하나만 정의된 인터페이스(람다식 사용의 필수 조건)

- 람다식은 메소드가 하나뿐인 인터페이스의 구현을 짧게 쓰는 것
<pre><code>
    // 전통 방식 사용
        Calculator c1 = new CalculatorImpl();
        System.out.println(c1.sumTwoNumber(1,2));

        // 익명 클래스 방식
        Calculator c2 = new Calculator() {
            @Override
            public int sumTwoNumber(int a, int b) {
                return a+b;
            }
        };

        // 람다식 사용
        Calculator c3 = (x, y) ->  x+y;
        System.out.println(c3.sumTwoNumber(10,20 ));

        // int와 int를 받아 int를 반환하는 것처럼 흔한 함수모양은
        // java.util.function 패키지에 표준 함수형 인터페이스로 준비되어 있다.

        // BiFunction<T, U, R> : T, U 두 인수를 받아 R 타입 반환
        BiFunction<Integer, Integer, Integer> sum = (x, y) -> x+y;
        System.out.println(sum.apply(100,200));

        // Consumer<T> : T를 받아 소비(처리)만 하고 반환 없음
        List<String> names = Arrays.asList("홍길동", "유관순", "이순신");

        for(String name : names){
            System.out.println(name);
        }

         Consumer<String> printName = name -> System.out.println("환영합니다. " + name + "님");
        // forEach :  리스트의 각 요소를 하나씩 꺼내 전달받은 Consumer에 넘겨준다.
        // printName에 담긴 람다식이 각 이름마다 실행됨
        names.forEach(printName);

        // 한줄로 줄이기
        names.forEach(name -> System.out.println("환영합니다 " + name + "님!"));

        // 외부 지역 변수 사용
        String prefix = "[회원]";
        names.forEach(name -> System.out.println(prefix + " " + name));

// prefix = "[고객]";
// 람다 안에서 사용하는 외부 지역 변수는 final 또는 effectively final 이어야 함
</code></pre>

## 표준 함수형 인터페이스

<pre><code>
    // 1. Consumer<T>: T 타입의 인자를 받아 '소비'만 한다(반환값 없음)
        Consumer<String> printer = message -> System.out.println(message);
        printer.accept("hello, Consumer");

        // 2. Supplier<T>: 아무것도 받지 않고, T 타입의 객체를 '공급'한다.
        Supplier<LocalDateTime> timeSupplier = () -> LocalDateTime.now();
        System.out.println("현재시간: " + timeSupplier.get());

        // 3. Function<T, R>: T 타입의 인자를 받아, R 타입의 객체로 '변환'하여 반환
        Function<String, Integer> length = str -> str.length();
        System.out.println(length.apply("hello, Function!"));

        Function<String, Integer> parseInt = Integer::parseInt;
        System.out.println(parseInt.apply("4123"));

        // 4. Operator 계열: 입력 타입과 반환 타입이 같은 Function 계열
        // UnaryOperator<T>: 동일한 T타입 인자를 1개 받아 같은 T 타입으로 반환
        UnaryOperator<String> toUpper = str -> str.toUpperCase();
        System.out.println(toUpper.apply("a"));

        // BinaryOperator<T> : 동일한 T 타입 인자를 2개 받아서 같은 T 타입으로 반환
        BinaryOperator<Integer> sumOperator = (num1, num2) -> num1 + num2;
        System.out.println(sumOperator.apply(2,3));

        // BiFunction<T, T, T> 의 특수 형태라 똑같이 apply 사용
//        System.out.println(sumOperator.apply(1000, 2999));


        // 5. Predicate<T>: T 타입의 인자를 받아 '검사'하고, boolean으로 '판단'하여 반환
        Predicate<String> isNotEmpty = str -> str != null && !str.isEmpty();
        System.out.println(isNotEmpty.test("Hello"));
        System.out.println(isNotEmpty.test(""));
        System.out.println(isNotEmpty.test(null));

</code></pre>

## Stream API

- 컬렉션에 들어있는 여러 데이터를 조건에 따라 거르고, 바꾸고 모으는 작업을 읽기 좋게 만들어주는 API
<pre><code>
    List<String> names = Arrays.asList("홍길동", "이민호", "유관순", "신사임당", "장보고", "강감찬", "이성계");

        // 기존 방식 : for-each문을 사용한 외부 반복
        for(String name: names){
            System.out.println(name);
        }
        System.out.println("==========================");

        // Stream 방식 - forEach를 사용한 내부 반복
        names.stream().forEach(name -> System.out.println(name));

        /* 이름이 3글자인 사람만 찾아서, "[홍길동]"과 같은 형태로 변환하여 출력 */
        for(String name: names){
            if(name.length() == 3){
                String decoratedName = "[" + name + "]";
                System.out.println(decoratedName);
            }
        }
        System.out.println("-----------------");

        names.stream()
                .filter(name -> name.length() == 3)
                .map(name -> "[" + name + "]")
                .forEach(decoratedName -> System.out.println(decoratedName));

        /* 결과물(새로운 List) 만들기
        * 성이 '이' 씨인 사람들을 찾아서, 알파벳 순서로 정렬한 후, 새로운 List로 만들어라 */

        List<String> leeFamily = names.stream()
                .filter(name -> name.startsWith("이"))
                .sorted(Comparator.reverseOrder())  // 가나다 역순으로 정렬
                .collect(Collectors.toList());   // 최종 결과물들을 새로운 List로 수집

        System.out.println("----------------");
        System.out.println(leeFamily);

        // 최종 연산
        // 1. count() : 조건에 맞는 요소의 개수 세기
        long leeCount = names.stream()
                .filter(name -> name.startsWith("이"))
                .count();   // 반환타입 long
        System.out.println(leeCount);

        // 2. anyMatch(): 조건에 맞는 요소가 하나라도 있는지 확인
        boolean hasJang = names.stream()
                .anyMatch(name -> name.equals("장보고"));  // 하나라도 조건 만족하면 true
        System.out.println(hasJang);

        // 3. allMatch(): 모든 요소가 조건을 만족하는지 확인
        boolean isAllThreeLetters = names.stream()
                .allMatch(name -> name.length() == 3);  // 모든 요소가 조건을 만족하면 true
        System.out.println(isAllThreeLetters);


</code></pre>
