# Spring

## IoC(Inversion of Control)

- 제어의 역전은 일반적인 프로그래밍에서 프로그램의 제어 흐름 구조가 뒤바뀌는 것을 의미한다.
  => 객체의 생성과 관리, 객체 간의 의존성 처리 등을 프레임워크에서 대신 처리해주는 것이 IoC의 대표적인 예이다.

## Bean

- Spring IoC Container에서 관리되는 객체를 말한다.
- 스프링은 Bean을 생성하고 초기화하고, 의존성 주입하고, 제거하는 등의 일을 IoC Container를 통해 자동으로 처리한다.
- 클래스 위에 @Compoenet, @Service, @Repository, @Controller 등의 어노테이션이 붙으면 Spring이 얘를 찾아서 객체로 만들어줌

## Bean 등록 방법

1. XML(요즘은 안씀)
2. JavaConfig

## Spring Container

- 객체를 만들고 보관하고 관리하는 공간
- 여기서 객체는 = Bean

## Dependency Injection(의존성 주입)

- Spring Container => Spring이 객체를 만들고 보관하고 관리하는 공간
- Bean => Spring Container가 관리하는 객체
- XML 설정 => java config의 Bean ComponentScan의 @Component
- 요청한 의존성을 Spring이 대신 주입해준다.

1.  Field 주입
    <code>

        @Autowired
        private BookDAO bookDAO;

    </code>

2.  Setter 주입
    <code>

        @Autowired
        private void setBookDAO(BookDAO bookDAO){
            this.bookDAO = bookDAO;
        }

    </code>

3.  Constructor 생성자 주입(가장 권장)
    <code>

        @Autowired
        public BookService(BookDAO bookDAO){
            this.bookDAO = bookDAO;
        }

## 전체 흐름

- Application 실행 -> Spring Container 생성 -> @Component 붙은 클래스들을 Bean으로 등록 -> @Autowired 보고 필요한 객체를 연결 -> getBean()으로 Bean 꺼내서 사용 -> 메서드 실행

- 핵심은 new를 없애는게 목적이 아니라, 객체 생성과 연결 책임을 Spring에게 맡기는 것

Servlet -> Service -> DAO -> JDBC
Controller -> Service -> Repository
