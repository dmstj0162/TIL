## 로그인 기능

📌 구현 방식

- 사용자 id, name은 테스트를 위해 하드코딩
- 전역 상태 관리 라이브러이 Zustand를 사용하여 로그인 상태 저장
- `persist`를 사용해 새로고침 시에도 로그인 상태 유지
  <br><br/>

📌 핵심 코드

<pre><code>
login: (user) => set({ user }), 
logout: () => set({ user: null }),

👉 사용자 정보를 전역 상태에 저장하고, 로그아웃 시 초기화
</code></pre>

<br><br/>
📌 상태 저장 구조

<pre><code>
persist(
  (set) => ({
    user: null,
    login: (user) => set({ user }),
    logout: () => set({ user: null }),
  }),
  { name: 'user-storage' }
)
👉 localStorage에 저장되어 페이지 이동/새로고침에도 유지됨
</code></pre>

<br><br/>
📌 조건부 렌더링 (로그인 상태 UI)

```jsx
{
  user ? (
    <div>
      <h3>{user.id}</h3>
      <p>{user.name}</p>
    </div>
  ) : (
    <p>유저 정보가 없습니다.</p>
  );
}
```

👉

로그인 상태 → 사용자 정보 표시<br>
비로그인 상태 → 안내 메시지 출력</br>
<br></br>
📌 추가 기능

- 로그아웃 시 user가 null이 되면서<br>
  👉 마이페이지 버튼이 보이지 않도록 처리</br>

<br></br>

## 2.마이페이지(내 프롬프트 조회)

📌 구현 방식

- 로그인된 사용자의 `userId`를 기준으로 데이터 요청
- 팀원이 만든 `PromptCard` 컴포넌트를 재사용
- 사용자별 프롬프트만 필터링해서 출력
  <br></br>

📌 핵심 데이터 요청

<pre>
<code>
fetch(`${BASE_URL}/prompts?userId=${Number(userId)}`)
👉 userId를 기준으로 해당 사용자 데이터만 조회
</code></pre>

📌 상태 관리 (프롬프트)

<pre>
<code>
set({ prompts: data })
👉 서버에서 받아온 데이터를 상태에 저장
</code></pre>

📌 데이터 흐름

1. 로그인 → user 저장 (Zustand)
2. 마이페이지 진입
3. userId 가져오기
4. API 요청 (/prompts?userId=...)
5. prompts 상태에 저장
6. 화면에 렌더링
   <br></br>

📌 핵심 로직 (중요 ⭐)

<pre><code>
useEffect(() => {
  if (!userId) return;
  if (prompts.length > 0) return;

  fetchPrompts(userId);
}, [userId]);

👉
userId 없으면 요청 안 함
이미 데이터 있으면 중복 요청 방지
userId 변경 시에만 fetch 실행
</code></pre>

<br></br>
📌 렌더링 (컴포넌트 재사용)

```jsx
{prompts.map((prompt) => (
  <PromptCard key={prompt.id} prompt={prompt} />
))}

👉
팀원이 만든 UI 컴포넌트를 재사용하여
데이터만 연결하는 방식으로 구현
```

## 발표 스크립트

안녕하세요.
저는 로그인 기능과 마이페이지에서 사용자별 프롬프트를 조회하는 기능을 구현했습니다.

먼저 로그인 기능입니다.
현재는 백엔드 인증이 완성되지 않은 상태라서, 사용자 id와 name을 임시로 하드코딩해서 사용했습니다. 대신 실제 서비스처럼 동작할 수 있도록 구조를 먼저 구현했습니다.

로그인된 사용자 정보는 Zustand를 사용해서 전역 상태로 관리했고, persist를 적용해서 새로 고침 이후에도 로그인 상태가 유지되도록 했습니다. 이렇게 저장된 user 정보는 다른 페이지에서도 사용할 수 있고, 로그아웃 시에는 user 값을 null로 바꿔서 마이페이지 접근이 안 되도록 처리했습니다.

다음은 마이페이지 기능입니다.
마이페이지에서는 로그인한 사용자가 작성한 프롬프트만 볼수 있도로 구현했습니다.

구현 방식은 간단하게, 전역 상태에 저장된 user의 userId를 가져와서 해당 id를 기준으로 서버에 요청을 보내고, 그 결과를 화면에 렌더링하는 구조입니다.

이때, API 요청은 useEffect를 사용해서 userId가 있을 때만 실행되도록 했고, 이미 데이터가 있는 경우에는 중복 요청을 하지 않도록 처리했습니다.

프롬프트 UI는 팀원이 구현한 PromptCard 컴포넌트를 재사용했고, 저는 그 컴포넌트에 사용자 데이터를 연결해서 로그인한 사용자에 맞는 데이터만 보여주도록 구현했습니다.

전체 흐름을 정리하자면,
로그인 -> user 상태 저장 -> 마이페이지 진입 -> userId로 데이터 요청 -> 화면 렌더링 이런 구조로 동작합니다.

이 기능을 통해 사용자별 데이터 분리와 상태 관리 흐름을 구현할 수 있었습니다.

## 자바 연산

<pre><code>
  ♠ 문자의 연산 
  System.out.println('a'+'b') 
  // 내부적으로 숫자 취급을 함(아스키코드와 유니코드)

  ♠ 문자열의 연산 
  System.out.println("hello"+"world")
  // 문자열과 문자열의 '+' 연산 결과는 문자열 합치기가 된다.
  // + 연산 외에 다른 연산은 사용하지 못함

  System.out.println("hello" + 123)
  // 다른 형태의 값과 + 연산도 문자열 합치기 결과가 나온다
  System.out.println("hello" + 'a') //helloa

  ♠ 논리 값의 연산 
  // 모든 연산자 사용 불가
  System.out.println(true + "a") // truea
  // 위 코드를 제외하고 다 불가  

</code></pre>
