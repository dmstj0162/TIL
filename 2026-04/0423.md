## useState

- 값을 저장하고, 바뀌면 화면도 다시 그리게 해주는 것
- 왜 필요하냐?
- 일반 변수는 값은 바뀌는데 화면은 안바뀌기 때문

<code>

    const [count, setCount] = useState(0);
    // count는 현재 값, setCount는 값을 바꾸는 버튼, 0은 처음 값

</code>

## useEffect

- 언제 실행할지 "타이밍 제어"

<code>

    useEffect(() => {
        console.log("처음 실행됨")
    }, [])
    // 언제 사용하냐
    // 페이지 처음 들어왔을 때
    // API 요청할 때
    // 값 바뀔 떄 실행

</code>

## useRef

- 값은 저장하는데, 화면은 안바뀜

<code>

    const inputRef = useRef(null);
    // 언제 쓰냐
    // input에 포커스 줄 때, 값은 기억해야하는데 렌더링은 안하고 싶을때

</code>

## useCallback

- 함수 재사용
  <code>

      const handleClick = useCallback(() => {
          console.log("클릭");
      }, [])
      // 언제 쓰냐
      // 컴포넌트 리렌더링 최적화할 때
      // 함수를 계속 새로 만들지 않게 함

</code>

## useContext

- 전역 상태 공유

<code>

    const user = useContext(UserContext);
    // 언제 쓰냐
    // 로그인 정보
    // 테마
    // 다크모드
    // props 없이 데이터 전달

</code>
