1. 로그인 + MyPage 구현 정리(zustand + next.js)

- zustand : 전역 user 상태 관리
- persist middleware : localStorage에 로그인 상태 저장
- Next.js App Router : 페이지 이동 및 렌더링

2. 로그인 상태 관리(zustand)

- useUserStore 구조
<pre>
<code>
    import { create } from "zustand";
    import { persist } from "zustand/middleware";

    export const useUserStore = create(
        persist(
         (set) => ({
        user: null,

      login: (user) => set({ user }),
      logout: () => set({ user: null }),
    }),
    {
      name: "user-storage", // localStorage key
    }
  )
);  
</code>
</pre>

* 핵심역할

- 로그인 시 user 저장
- 로그아웃 시 user 제거
- persist로 새로고침 후에도 유지

3. 로그인 흐름
   로그인 버튼 클릭
   -> login(user)
   -> zustand 상태 변경
   -> localStorage 자동 저장(persist)

5. MyPage 문제 해결 과정

- 발생문제

* 로그인 후 마이페이지 이동 ok
* 뒤로가기 시 로그인 풀린 것처럼 보임
* 새로고침 시 잠깐 로그인 풀린 것처럼 보임

- 원인

* persis 복구 전에 React가 먼저 렌더링됨
* 초기값(user=null)이 먼저 화면에 표시됨
* hydration 타이밍 문제로 UI 깜빡임 발생

6. 해결 방법
user가 이미 있는 경우 UseEffect 되지 못하도록 if문 추가
<pre>
<code>
  useEffect(() => {

    const getUser = async () => {
      if (!user) {
        try {
          const data = await fetchUsers();
          if (data)
            login(data);
        } catch (error) {
          console.error('유저불러오기실패', error)
        }
      }
      getUser();
    }
  }, []);
</code>
</pre>
