1. useEffect
   - 함수형 컴포넌트에서 렌더링 된 이후 시점에서 수행할 내용이 필요한 경우, 사용할 수 있는 기능을 hooks로 제공해주고 있는 useEffect이다.
   - useEffect는 첫 마운트 직후에는 무조건 한 번 실행된다.
     1-1. 의존성 배열 생략 : 매 렌더링마다 실행
     1-2. 빈 의존성 배열 [] : Mount(첫 렌더링) 시에만 실행, 주로 초기 데이터 로딩(API 호출) 등에 사용
     1-3. 의존성 배열에 특정 값 지정 [count]: 해당 값이 변경될 때만 실행
   - Cleanup 함수: useEffect가 반환하는 함수이다. 컴포넌트가 화면에서 사라지기 직전(Unmount)에 호출된다.
   <pre>
   <code>
   useEffect(
               () => { console.log('useEffect 동작') }, // 수행할 동작 - 콜백 함수
               [] // 의존성 배열을 빈 배열로 설정하면 업데이트 시점에는 동작하지 않는다.
           )</code>
   </pre>

2. mount
   - React는 컴포넌트를 만들고 그것을 브라우저 화면(DOM)에 붙이는데 이 붙이는 첫 순간이 바로 mount이다.
   - React 흐름
     - Mount : 처음 나타날 때
     - Update : 상태(state)나 props가 바뀔 때
     - Unmount : 화면에서 사라질 때

3. useMemo
   - React는 상태(state)나 props가 바뀌면 컴포넌트를 다시 렌더링함 -> 이때 매번 무거운 계산을 다시 하면 성능이 낭비될 수 있음
     => 그래서 useMemo로 결과를 캐싱(기억)해둠

   <pre>
   <code>
       const memoizedValue = useMemo(() => {
           return 계산결과;
       }, [의존성]);
   </code>
   </pre>
   - 첫 렌더링 -> 계산 실행
   - 이후 렌더링 -> 의존성이 바뀔 때만 다시 계산
   - 안 바뀌면 -> 이전 결과 그대로 사용
   * 좋은 사용 케이스
   - 정렬, 필터링 같은 비용 큰 연산
   - 배열/객체를 만들어서 불필요한 리렌더 막고 싶을 때
   - props로 내려주는 값이 매번 새로 생성되는 걸 막고 싶을 때

4. useCallback
   - 함수 자체를 기억함
   - 함수 내용이 아니라, 함수 객체를 유지
   - 조건이 안바뀌면 -> 같은 함수 계속 사용
   * 왜 함수까지 기억해야 하냐?
   - React에서는 함수를 새로 만들면 = 다른 값으로 취급되기 때문

5. useRef
   - 렌더링과 상관없이 값을 계속 들고 있는 상자
   - 값을 저장하지만, 바뀌어도 리렌더링을 일으키지 않음
   - 그리고 DOM 요소를 직접 참조할 때도 사용
     => 값 바꿔도 렌더링 안 됨, 값이 계속 유지됨(실제 값은 .current에 들어있고, 값을 바꿔도 화면을 다시 그리지 않음)
   - DOM 직접 접근
    <pre>
   <code>
       import { useRef } from "react";
   
        function InputFocus() {
        const inputRef = useRef(null);
   
        const focus = () => {
            inputRef.current.focus();
        };
   
        return (
            <>
             <input ref={inputRef} />
             <button onClick={focus}>포커스</button>
             </>
         );
        }
   </code>
   </pre>

6. custom
   - 이름이 use로 시작하는 일반 자바스크립트 함수
   - 내부에서 useState와 useEffect를 마음껏 사용할 수 있다.
    <pre>
   <code>
      const { useState, useEffect } = React;
   
        function useDelayTmer(delay) {
            // 시간이 지났는지 여부를 저장하는 상태
            const [isReady, setIsReady] = useState(false);
   
            useEffect(() => {
                console.log(`${delay}ms 타이머 시작`);
   
                const timer = setTimeout(() => {
                    setIsReady(true);
                    console.log('타이머 종료')
                }, delay)
   
                // 언마운트 될 때 타이머 정리(메모리 누수 방지)
                return () => clearTimeout(timer)
            }, [delay])
   
            return isReady;
   
        }
   
        function App() {
   
            const isReady = useDelayTmer(3000);
            return (
                <>
                    <h1>Custom Hook</h1>
                    <hr />
                    {isReady ? <h2 style={{ color: 'blue' }}>준비완료</h2> : <h2 style={{ color: 'red' }}>잠시만 기다려주세용</h2>}
                </>
            )
        }
   </code>
   </pre>
   7. 콜백함수 지옥에서 벗어나오기 위해서 Promise, async function, await을 사용
   - 동기식(synchronous) : 순섣대로 하나씩 실행(기다림ㅇ)
   - 비동기식(Asynchronous) : 기다리지 않고 다음 작업 진행(기다림 X)

   - Promise : 비동기 작업이 미래의 언젠가 끝날텐데, 작업이 끝나면 성공(resolve)했는지 실패(reject)했는지 꼭 알려줄게! 하고 약속하는 '약속 증서' 객체를 먼저 반환하는 개념이다.

    <pre>
    <code>
        function increase(number) {
            const promise = new Promise((resolve, reject) => {
                setTimeout(() => {
                    const result = number + 10;
   
                    if (result > 50) {
                        // 실패 -> reject 함수 호출하며 에러 전달
                        const e = new Error('숫자가 너무 큽니다.');
                        return reject(e);
                    }
                    // 성공 -> 계산된 결과와 함께 resolve 처리
                    resolve(result);
                }, 1000)
            })
            return promise;
        }
   
        async function run() {
            console.log('작업시작 ...');
            // await: increse(0) Promise가 끝날 때까지 기다렸다가,
            //         resolve된 결과값을 result 변수에 할당
   
            try {
   
                let result = await increase(0);
                console.log(result);
                result = await increase(result);
                console.log(result)
                result = await increase(result);
                console.log(result)
                result = await increase(result);
                console.log(result)
                result = await increase(result);
                console.log(result)
                result = await increase(result);
                console.log(result)
                result = await increase(result);
                console.log(result)
            } catch (error) { // try 블록 안에서 reject 되면 catch 블록 실행 
                console.log(error)
            }
            console.log('모든 작업이 끝났습니다.')
        }
        run();
   
    </code>
    </pre>
